# MySQL Database Skill

## 描述
本技能允许 Agent 与 MySQL 数据库进行交互。Agent 可以执行只读查询以获取数据，查询数据库结构，或在授权情况下执行变更操作。设计上融合了防御性策略，防止资源耗尽和恶意操作。

## 能力
- **查询数据**: 执行 SELECT 语句检索信息（受行数限制保护）。
- **结构探查**: 列出数据库中的表，或描述特定表的结构。
- **执行变更**: 执行 INSERT, UPDATE, DELETE 语句 (受安全策略约束)。

## 使用场景
- 用户询问关于数据库中的业务数据（例如：“有多少用户注册了？”）。
- 用户需要了解数据表结构以进行后续操作。
- 用户需要对数据进行简单的增删改操作。

## 入口函数
所有功能通过 `scripts.actions` 模块中的函数暴露。

## 动作列表

### 1. `run_sql_query`
执行一条 SQL 语句。
*   **参数**: `sql` (string) - 有效的 SQL 语句。
*   **返回**: JSON 格式的查询结果或执行状态。
*   **约束**: 
    - SELECT 语句默认受最大行数限制（防止内存溢出）。
    - 危险操作（如 DROP, TRUNCATE）将被拦截。

### 2. `get_table_schema`
获取指定表的结构信息。
*   **参数**: `table_name` (string) - 目标表名称。
*   **返回**: 包含字段名、类型、键信息的 JSON 列表。

### 3. `list_tables`
列出当前数据库中的所有表。
*   **参数**: 无。
*   **返回**: 表名列表。

## 安全警告
- 对于写入操作，Agent 应当在执行前向用户确认。
- 禁止执行破坏性极强的命令（如 DROP DATABASE）。
